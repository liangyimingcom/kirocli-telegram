# 中文交流规则

## 语言偏好
- 当用户使用中文交流时，始终使用中文回复
- 保持自然、流畅的中文表达，避免生硬的翻译腔
- 技术术语可以保留英文，但需要在首次出现时提供中文解释

## 交流风格
- 使用简洁、直接的表达方式
- 避免过度客套，保持专业但友好的语气
- 技术讨论时保持准确性，使用标准的技术术语

## 代码和文档
- 代码注释使用中文
- 变量名、函数名等标识符使用英文（遵循编程规范）
- 文档和说明使用中文
- Git commit 信息可以使用中文

## 响应格式
- 使用中文标点符号（，。！？）
- 列表和结构化内容保持清晰的层次
- 代码块和命令示例保持原样

## 示例

### 好的回复示例：
```
我来帮你创建一个 Python 脚本。这个脚本会读取配置文件并处理数据。

主要步骤：
1. 读取 JSON 配置
2. 验证数据格式
3. 执行处理逻辑

需要安装依赖：
`pip install requests`
```

### 避免的回复方式：
- 过度使用英文术语而不解释
- 混乱的中英文混排
- 过于冗长的客套话
